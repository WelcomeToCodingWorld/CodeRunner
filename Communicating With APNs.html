Communicating With APNs

The APNs provider API lets you send remote notifications to your app on iOS, tvOS, and macOS devices, and to Apple Watch via iOS. The API is based on the HTTP/2 network protocol. Each interaction starts with a POST request, containing a JSON payload, that you send from your provider server to APNs. APNs then forwards the notification to your app on a specific user device.(APNs服务商的API让你能够发送远程通知给你iOS，tvOS和macOS上的app，这些API是基于http／2网络协议的。每一个交互都从一个Post请求开始，该请求包含一个要你们的服务器要发送给APNs的JSON格式的payload，然后APNs把通知转发给你的app用户的特定设备上。)

Provider Authentication Tokens(提供身份验证token)

To securely connect to APNs, you can use provider authentication tokens or provider certificates. This section describes connections using tokens.(为了安全连接到APNs,你可以用服务提供者的身份token或者证书。)
